# AllOfSoccer

## 프로젝트 비전

아직도 많은 축구팀들이 네이버/다음 카페에서 PC로 경기 매칭 글을 올리고, 이전 글을 복붙하며 번거롭게 상대팀을 찾고 있다.

**AllOfSoccer는 이 과정을 모바일로 옮긴다.**

- 몇 번의 터치만으로 매칭 글 등록
- 이전에 올렸던 매칭 정보 재사용
- 날짜/지역/경기종류 필터로 빠른 상대팀 탐색
- 용병 모집까지 한 곳에서 해결

## 현재 구현 상태

### 완성도 요약

| 화면 | 상태 | 비고 |
|---|---|---|
| 로그인 | 20% | Apple 로그인 UI만, 서버 연동 없음 |
| 매칭 리스트 | 85% | 필터/캘린더 완성, Mock 데이터 사용 중 |
| 매칭 상세 | 70% | UI 완성, 참가 신청 미구현 |
| 팀 모집 등록 | 50% | UI 완성, 서버 저장 없음 |
| 용병 모집 등록 | 40% | UI 부분 구현, 서버 저장 없음 |
| 마이페이지 | 25% | 메뉴 네비게이션만 구현 |
| 채팅/실시간 | 0% | 미착수 |
| 푸시 알림 | 0% | 미착수 |

### 구현된 핵심 기능
- 수평 캘린더 날짜 선택 (90일)
- 필터링: 지역(서울북부/남부, 경기북부/남부, 인천/부천, 기타), 경기종류(11vs11, 풋살), 매칭여부
- 매칭 리스트 테이블뷰 + 셀 디자인
- 좋아요 토글
- 매칭 생성 입력 UI (2단계: 기본정보 -> 상세정보)
- 네트워크 모델 설계 완료 (Match, Team, User, Pagination)

### 부족한 부분
- 모든 API 호출이 Mock 데이터 또는 미구현
- 매칭 생성 시 서버 저장 불가
- 로그인 후 토큰 관리/세션 유지 없음
- 에러 처리, 로딩 상태, 유효성 검사 전반적 미흡

## 개발 로드맵

### Phase 1 - 핵심 플로우 완성 (iOS 우선)

> 서버 작업 전, iOS 앱의 UI/UX와 클라이언트 로직을 먼저 완성한다.

**1-1. 매칭 등록 플로우 완성**
- [ ] 팀 모집 등록: 입력값 검증 + 데이터 모델 바인딩
- [ ] 용병 모집 등록: 입력값 검증 + 데이터 모델 바인딩
- [ ] "이전 매칭 정보 불러오기" 기능 구현 (카페 복붙 문제 해결의 핵심)
- [ ] 등록 완료 확인 화면
- [ ] 등록 후 매칭 리스트 자동 갱신

**1-2. 매칭 리스트 보완**
- [ ] 정렬 기능 로직 구현 (최신순, 가격순, 거리순)
- [ ] 가격대 필터 로직 연결
- [ ] Pull to Refresh
- [ ] 페이지네이션 (무한 스크롤)
- [ ] 빈 상태 화면 (검색 결과 없음)

**1-3. 매칭 상세 화면 완성**
- [ ] 참가 신청 버튼 동작
- [ ] 참가자 목록 표시
- [ ] 공유 기능 (카카오톡, 링크 복사)
- [ ] 신고/차단 기능

**1-4. 로그인/인증 완성**
- [ ] Apple 로그인 서버 연동
- [ ] 토큰 저장 및 자동 로그인
- [ ] 토큰 만료 시 갱신
- [ ] 로그아웃
- [ ] 비로그인 상태 접근 제한 처리

### Phase 2 - 사용자 관리

**2-1. 마이페이지**

- [ ] 활동 기록/통계

**2-2. 팀 관리**
- [ ] 팀 생성 (팀명, 로고, 소개, 활동 지역)
- [ ] 팀 정보 수정
- [ ] 팀원 관리 (초대, 제거)
- [ ] 팀 목록 조회

**2-3. 알림**
- [ ] 앱 내 알림 센터
- [ ] 매칭 상태 변경 알림 (모집완료, 취소 등)
- [ ] 참가 신청 알림
- [ ] 푸시 알림 (APNs)

### Phase 3 - 차별화 기능

**3-1. 빠른 재등록**
- [ ] 이전 매칭 템플릿 저장
- [ ] 한 번의 터치로 동일 조건 재등록

- [ ] 경기장 즐겨찾기
- [ ] 날씨 정보 연동

## 프로젝트 구조

```
iOS/AllOfSoccer/
├── Application/          # AppDelegate, SceneDelegate
├── Login/                # Apple Sign-In 로그인
├── Main/                 # 매칭 리스트 (메인 화면)
│   └── GameMatchingSub/  # 캘린더, 필터 컴포넌트
├── Recruitment/          # 매칭 등록
│   ├── TeamRecruitment/  # 팀 모집 (2단계)
│   ├── ManRecruitment/   # 용병 모집 (2단계)
│   ├── MercenaryMatch/   # 용병 매칭
│   └── Component/        # 등록 공용 컴포넌트
├── GamematchingDetail/   # 매칭 상세
├── MyPage/               # 마이페이지
├── Network/              # API 통신, 모델
├── Components/           # 공용 UI 컴포넌트
├── Extension/            # Swift 확장
├── Base/                 # BaseViewController
└── SupporttingFiles/     # Info.plist 등
```

## 핵심 데이터 모델

```
Match (매칭)
├── 기본: id, title, description
├── 일시: date
├── 장소: location, address, latitude, longitude
├── 경기: matchType(6v6/11v11), genderType, shoesRequirement
├── 조건: ageRangeMin/Max, skillLevel
├── 비용: fee
├── 참가: maxParticipants, currentParticipants
├── 상태: status, isOpponentMatched, hasFormerPlayer
├── 용병: mercenaryRecruitmentCount
└── 팀: Team (name, logoUrl, captain)
```
